# Getting started

Welcome to Clairdash! We're going to try to get you developing, exploring or just browsing your data in no time.

We've put together a couple of tutorials to get people started with working in Clairdash. To decide which one's the right fit for you, just read the descriptions below:

## 🏗 Setting up a new project

If you:

✅ are starting a new Clairdash project from scratch
✅ have a Clairdash project connected, but it's empty/ you still need to set up your tables
✅ know how to use dbt

then this tutorial is for you! [🏗 Setting up a new project](/get-started/setup-clairdash/intro)

## 👩‍💻 Developing in Clairdash

If you:

✅ have an existing Clairdash project with some tables and dimensions
✅ know how to use dbt
✅ want to add or change some dimensions, metrics, or tables in your project
✅ want to add formatting or descriptions to existing metrics, dimensions or tables in your project

then this tutorial is for you! [👩‍💻 Developing in Clairdash](/get-started/develop-in-clairdash/intro)

Also check out our [Clairdash Developer Quickstart](https://clairdash.notion.site/Clairdash-developer-quickstart-3192f80173ef4f0bbbe0f0f88b726a76), which can be copied and customized for you or other Clairdash developers in your organization.

## 🔭 Learning to explore data in Clairdash

If you:

✅ want to explore data in Clairdash
✅ want to learn how to build charts + dashboards in Clairdash
❌ are **not** someone who uses dbt  
❌ aren't responsible for maintaining your metrics + dimension definitions

then this tutorial is for you! [🔭 Learning to explore data in Clairdash](/get-started/exploring-data/intro)
